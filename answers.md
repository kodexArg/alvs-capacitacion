## Bloque A — Audiencia y logística

- **A1. Cantidad de personas:** ~4 personas (grupo pequeño)
- **A2. Perfil del grupo:** Jefes / alta dirección. Alto poder de decisión. Consumen dashboards e información. Muy poco o nulo conocimiento de AI.
- **A3. Idioma:** Español rioplatense (presencial)
- **A4. Modalidad:** Presencial. Monitor + notebook del instructor. Las 4 personas miran desde sus lugares — **sin laptops propias**. Todo el hands-on lo ejecuta el instructor en pantalla.
- **A5. Dispositivos:** Monitor como pantalla principal, notebook del instructor como única máquina activa.
- **A6. Restricciones IT / redes:** Red corporativa libre (claude.ai y similares accesibles)
- **A7. Fecha estimada:** MAÑANA (2026-04-14) — urgente

## Bloque B — Herramienta principal

- **B1. Plataforma elegida:** **Claude Desktop** (app Windows/Mac) como protagonista. También mostrar Gemini (web), ChatGPT (web), posiblemente DeepSeek como panorama comparativo.
- **B2. Licencias corporativas existentes:** Ninguna (solo free tiers)
- **B3. Mostrar una o varias herramientas:** Varias — Claude Desktop como primaria, otras como tour rápido

## Bloque C — Alcance pedagógico

- **C1. Temas adicionales a incluir:** ética básica / privacidad de datos corporativos, limitaciones y alucinaciones, casos de uso para su rol de alta dirección
- **C2. Framework de prompting:** CQR (Contexto / Consulta / Resultado esperado) como nombre principal. Mostrar también panorama de otros frameworks (RACE, CO-STAR) para contexto.
- **C3. System prompts:** Enseñar el MÉTODO — ellos construyen el suyo verbalmente durante la clase, el instructor ejecuta en pantalla.
